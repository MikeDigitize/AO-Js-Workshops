A quick run through of WebPack

What is Webpack?

It's a JavaScript module bundler

But can be expanded to bundle any type of file

And perform all manner of processing to those files

Think of Gulp

but Gulp works on a file level

Whilst Webpack works on a project level

Tell you more you say?

Sure...

So what can Webpack do exactly?

Compile and bundle your assets

That might not mean much to you

It compiles and bundles your JavaScript

Intelligently


// ES6 import
import React from "react";

// Node / CommonJS
var React = require("react");

Webpack identifies and eliminates dead code that's not being used

It can identify common dependencies such as React or jQuery

And makes cacheable shared bundles to prevent code duplication

What else does Webpack do?

It does some really useful stuff

  • Concatenating files
  • Minifiying files
  • Compiling SASS
  • Transpiling ES6 to ES5
  • Converting JSX to React

Through the use of super fast loaders

Which makes it extremely modular

Plus it does some unusual stuff too

  • It has it's own browsersync-like dev server
  • Async loading of assets for single paged apps
  • React hot loading for live editing of components without losing state
  • Localising CSS

It doesn't have to be used as a standalone tool either

It can be used in Gulp like any other Gulp plugin

Or ran from your own custom script in Node

And all these features run on a project level

So you just give WebPack a single entry point

And it'll take care of the rest for you

Sounds pretty cool doesn't it?

But wait, is there any bad stuff?

The documentation

First time users trying to get started by following the docs often feel like

But never fear, here's some tips for getting started

Prequisites to using Webpack

You need to have

  • NodeJS installed
  • And Webpack installed globally
npm install webpack -g

That's it!

The Webpack config

In your root create a file called webpack.config.js

It's just an object that uses Node's module.exports syntax


module.exports = {
    entry : "./path/to/your/entry/file",
    output : {
        filename : "./path/to/your/output"
    },
    watch : true // watch for changes
};

Example dependency file


module.exports = function(desc) {
    return "Yo' moma's soooo " + desc;
};

Example entry file


var yomama = require("./yomama");
console.log(yomama("fat"));

Then from the command line type

webpack

and hit return

Yo' moma's soooo fat

Winning!

Using ES6

Webpack uses loaders to process files

npm install --save-dev babel-loader babel-core babel-preset-es2015

Create a ".babelrc" file in your root and add this object


{ "presets": ["es2015"] }

Add this loader to your config


module: {
    loaders: [{
        test: /\.js$/,
        exclude: /node_modules/,
        loader: 'babel-loader'
    }]
};

Example dependency file


export let yomama = (desc) => `Yo' mama's sooooo ${desc}`;

Example entry file


import { yomama } from "./yomama.js";
console.log(yomama("fine"));
Yo' mama's sooooo fine

So that's a quick taste of WebPack

There's a ton of awesome loaders to play with

So go experiment and see what it can do

It's pretty sweet
