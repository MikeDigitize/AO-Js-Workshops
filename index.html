A quick run through of WebPack

What is Webpack?

It's a JavaScript module bundler

But can be expanded to bundle any type of file

And perform all manner of processing to those files

Think of Gulp

but Gulp works on a file level

Whilst Webpack works on a project level

Tell you more you say?

Sure...

So what can Webpack do exactly?

Compile and bundle your assets

Does that even mean anything to you?

It handles your JavaScript modules

Intelligently


// ES6 import
import React from "react";

// Node / CommonJS
var React = require("react");

Webpack identifies and eliminates dead code that's not being used

It identifies common dependencies such as React or jQuery

And makes cacheable shared bundles to prevent code duplication

What else does Webpack do?

It does all the usual stuff

  • Concatenating
  • Minification
  • Compiling SASS
  • Transpiling ES6
  • Converting JSX to React
  • Super fast file watching and compiling

Through the use of loaders

Which makes it extremely modular

Plus it does some unusual stuff too

  • Async loading of assets for single paged apps
  • Optimising image files
  • Converting and inlining CSS @font-face rules
  • Localising CSS

Plus: it plays beautifully with React and Redux too

And it doesn't have to be used as a standalone tool either

It can be used in Gulp like any other Gulp plugin

Or ran from your own custom script in Node

And it does all this on a project level

So you just give it a single entry point

And it'll take care of the rest for you

Sounds pretty cool doesn't it?

But wait, is there any bad stuff?

The documentation

First time users trying to get started by following the docs often feel like this

But never fear, here's some tips for getting started

Prequisites to using Webpack

You need to have

  • NodeJS installed
  • And Webpack installed globally
  • That's it!

You can also install webpack-dev-server globally (not required, just a nice to have) which will restart your app every time you make a save

The Webpack config

In your root create a file called webpack.config.js

It's just an object that uses Node's module.exports syntax


module.exports = {
    entry : "./src/path/to/your/entry/file",
    output : {
        filename : "./build/path/to/your/output"
    },
    watch : true // watch for changes
};

Example entry file


var React = require("react"); // you've installed React locally via npm
var ReactDOM = require("react-dom");
var App = require("./src/path/to/components/app");

ReactDOM.render(< App />, document.getElementById("app-holder"));

From the command line

Just type


webpack

and hit return

Winning!

Webpack's loaders


module.exports = {
    // ...stuff omitted
    module: {
        loaders: [{
            test: /\.js$|\.jsx$/,
            exclude: /node_modules/,
            loader: 'babel-loader'
        }]
    }
};
